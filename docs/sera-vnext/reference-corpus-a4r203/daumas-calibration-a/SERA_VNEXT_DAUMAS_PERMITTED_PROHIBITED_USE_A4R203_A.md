# SERA vNext Daumas Permitted and Prohibited Use - A4R203-A

Date: 2026-06-04
Phase: A4R203-A
Status: GOVERNANCE_REGISTER_ONLY

## 1. Permitted use

- methodological reference
- applied human-analysis depth reference
- evidence-depth checklist source
- reasoning-pattern source
- support for candidate-only review design
- support for author review preparation
- support for governance simplification
- support for synthetic guardrail design

## 2. Prohibited use

- factual source substitution for external real events
- automatic reentry trigger
- automatic baseline trigger
- automatic fixture trigger
- selectedCode justification
- releasedCode justification
- finalConclusion trigger
- READY trigger
- product shortcut
- substitute for official source locator discipline

## 3. Error examples

Incorrect:

- using Daumas to decide a final Colgan boundary
- using Daumas to fill missing factual data for G-WNSB or Execuflight
- using Daumas to promote a candidate into a fixture
- using Daumas to declare a releasedCode path

Acceptable:

- using Daumas to ask whether cue, goal, action, and context are deep enough
- using Daumas to decide that a case still needs hold or re-audit
- using Daumas to design a better author-review checklist
- using Daumas to define what a synthetic should include before any later authorization

## 4. Governance statements

- Hendy remains the primary source for causal logic.
- Daumas remains the applied methodological and human-depth reference.
- Official event sources remain the factual source for real-event evidence.
- Daumas cannot authorize broader reuse without explicit human review.
- Governance simplification should come before any synthetic materialization.
