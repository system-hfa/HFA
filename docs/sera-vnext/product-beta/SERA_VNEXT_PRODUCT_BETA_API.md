# SERA vNext Product Beta API

Internal endpoints:

- `POST /api/admin/sera-vnext/analyses`
- `GET /api/admin/sera-vnext/analyses`
- `GET /api/admin/sera-vnext/analyses/:id`
- `POST /api/admin/sera-vnext/analyses/:id/reanalyze`
- `POST /api/admin/sera-vnext/analyses/:id/reviews`
- `POST /api/admin/sera-vnext/analyses/:id/archive`
- `POST /api/admin/sera-vnext/analyses/:id/restore`
- `GET /api/admin/sera-vnext/analyses/:id/export`

All endpoints require `SERA_VNEXT_PRODUCT_BETA_ENABLED=true`, bearer auth, and enterprise admin authorization. Responses are no-store and include `x-request-id` when generated by the handler.
