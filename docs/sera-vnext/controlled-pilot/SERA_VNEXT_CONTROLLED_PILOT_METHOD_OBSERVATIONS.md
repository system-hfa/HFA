# SERA vNext Controlled Admin Pilot — Method Observations

## Protected Methodology Status

- The causal SERA baseline was not reopened.
- The v0.1 engine was not modified during this phase.
- Canonical tree questions, baseline fixtures, expected outputs, and Product Alpha locks remained protected.
- All findings below are product and reviewer-utility observations, not methodology changes.

## Observations From 10 Real Pilot Cases

| Observation          | Count | Interpretation                                                            |
| -------------------- | ----- | ------------------------------------------------------------------------- |
| Escape point useful  | 8     | Useful in most cases, but still needed human attention in boundary cases. |
| P/O/A useful         | 0     | Current candidate P/O/A is not yet actionable enough for reviewers.       |
| Preconditions useful | 1     | Current precondition surfacing is usually too weak or implicit.           |
| Uncertainty clear    | 10    | Reviewers could understand uncertainty boundaries.                        |
| Warnings clear       | 10    | Warnings presentation was understandable in the pilot.                    |

## Recurrent Pilot Issues

| Issue                              | Count |
| ---------------------------------- | ----- |
| candidate_poa_not_actionable       | 10    |
| preconditions_not_explicit         | 9     |
| escape_point_needs_human_attention | 2     |

## Interpretation

- The evidence-first and non-final workflow is operable for internal admin use.
- Escape point output is the strongest current reviewer aid.
- Candidate P/O/A and preconditions need UX and output-quality refinement before any broader cohort.
- The limitations are consistent with a controlled continuation decision and do not justify reopening methodology.
