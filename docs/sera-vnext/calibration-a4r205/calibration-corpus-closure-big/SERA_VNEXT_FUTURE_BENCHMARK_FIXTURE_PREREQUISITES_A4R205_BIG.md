## 3. Future Fixture Design Prerequisites

No future candidate fixture should be considered unless all of the following are satisfied:

1. the candidate case has stable evidence and a reproducible source trail.
2. there is no unresolved escape-point ambiguity at the intended fixture boundary.
3. there is no unresolved high agent-migration pressure at the intended direct-actor boundary.
4. the expected behavior is clear enough to validate without inventing final causal truth.
5. adversarial validation has already tested consequence quarantine, actor retention, and negative-control pressure.
6. explicit human approval exists for fixture-design consideration.
7. there is no active downstream dependency expecting the case to behave like released methodology.

Cases that currently fail these future fixture prerequisites:

- `Asiana 214`, if the automation warning is stripped away.

## 4. Future Baseline Rules

Any future baseline gate would require:

1. separate human authorization after benchmark and fixture design work is closed.
2. a dedicated baseline gate that is distinct from corpus closure and distinct from fixture design.
3. proof that non-final candidate-only records are not being reinterpreted as released truth.
4. proof that synthetic support and human-applied input remain quarantined appropriately.

Therefore:

- baseline is not an active next phase;
