# SERA vNext Archive Policy

This archive stores historical artifacts that are not active methodology sources.

Files under `docs/sera-vnext/archive/**`:
- are not active SERA sources;
- must not be used for front-end calibration;
- must not be used for reference-case construction or validation;
- must not be used for author approval evidence;
- may be read only as historical/audit record.

Archive folders:
- `invalid-methodology/`: contaminated or noncanonical methodology artifacts.
- `superseded/`: replaced by newer controlled artifacts.
- `exploratory/`: experimental notes not approved as active source.
- `pre-canonical/`: material created before canonical-tree lock and no longer active.
