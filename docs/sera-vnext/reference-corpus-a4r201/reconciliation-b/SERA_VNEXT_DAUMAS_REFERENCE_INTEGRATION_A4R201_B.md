# SERA vNext Daumas Reference Integration A4R201-B

Status: HUMAN_METHOD_REFERENCE_ONLY
Phase: A4R201-B

## Role

Daumas is a human and methodological reference for operationalizing human factors analysis in offshore/MDC contexts. It is not a factual source for any real accident or incident in the A4R201-B reconciliation.

## What To Compare With Daumas

Compare whether candidate event material has enough decision-context detail to support later human review:

- temporal sequence before the candidate escape point;
- actor role and PF/PM separation where relevant;
- cues, warnings, callouts, and procedural context;
- evidence available before or at the candidate escape point;
- separation of factual evidence from external analysis and conclusions.

## Use For POA Calibration Without Automatic Reentry

Daumas may help check whether a future source slice has the kind of human-work detail needed for later POA calibration. It does not supply a POA answer, does not close a canonical-tree branch, and does not authorize automatic reentry of held candidates.

## Use For Source-Depth Review

Daumas can be used as a depth yardstick: if an event record only has a short narrative, no actor-bound sequence, no warnings/callouts, and no procedure context, it is not deep enough for SERA reference work even if the event is famous.

## Use For Synthetic Fill

Daumas can support synthetic fill only at the methodological-design level, after real-event evidence has exposed a gap that cannot be safely covered by available reports. Synthetic material must remain separated from real-event records.

## Limits

- Daumas is not a primary factual source for real events.
- Daumas does not promote READY.
- Daumas does not create selectedCode, releasedCode, or finalConclusion.
- Daumas does not create fixture, baseline, product, or downstream status.
- Daumas does not override Hendy/SERA as the protected causal baseline.
