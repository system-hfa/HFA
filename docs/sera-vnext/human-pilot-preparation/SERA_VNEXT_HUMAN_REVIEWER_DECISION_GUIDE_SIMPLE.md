# SERA vNext Human Reviewer Decision Guide

## ACCEPT_AS_WORKING_HYPOTHESIS

Use this when the escape point is plausible, the direct actor is supported, P/O/A candidates are useful, preconditions are separated from the escape point, and no critical evidence misuse is present.

## REJECT_WORKING_HYPOTHESIS

Use this when the proposed escape point is a consequence, post-escape evidence is used as cause, the actor is unsupported, P/O/A does not follow from the evidence, or a strong methodological error makes the hypothesis unusable.

## REQUIRES_MORE_EVIDENCE

Use this when the narrative does not support a defensible review of actor, timing, perception, objective, action, automation mode, callout, or first useful escape point.

## RETURN_FOR_REANALYSIS

Use this only after the input narrative is corrected or new factual evidence is added. This is not a substitute for rejection when the current facts are enough to identify a bad hypothesis.
