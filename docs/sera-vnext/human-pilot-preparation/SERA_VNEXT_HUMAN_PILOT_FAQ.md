# SERA vNext Human Pilot FAQ

## Is this a final classification?

No. This is a candidate-only working hypothesis for human review.

## Can I use it for operational decisions?

No. The pilot is internal and non-operational.

## Can I change code during the pilot?

No. Record issues in the result template and issue backlog. Code changes require a separate authorized phase.

## What if I disagree with the engine?

Use the review decision that fits the evidence and explain the reason in comments.

## What if evidence is missing?

Use `REQUIRES_MORE_EVIDENCE`.

## What if the escape point seems wrong?

Reject the working hypothesis or request more evidence, depending on whether the current facts are sufficient.

## What is a precondition?

A precondition is context or a candidate contributing condition that helps explain why the unsafe act or condition became more likely. It is not automatically the escape point.

## What is post-escape evidence?

Post-escape evidence is information that appears after the first useful escape point. It can help understand the event but must not be used to create the initial causal basis.

## Why is consequence not cause?

The outcome occurs after the causal opportunity. The review must anchor on the first useful escape point and the direct actor evidence at that point.

## What does candidate-only mean?

It means the output is a review aid. It does not release final SERA codes, final conclusions, or downstream outputs.

## What should I report as a bug?

Report missing non-final warnings, broken navigation, inability to submit review, export failure, tenant access issue, audit gap, misleading labels, or evidence shown in the wrong section.

## What must not be entered?

Do not enter real client data, credentials, protected operational details, personal names, email addresses, or confidential attachments.
